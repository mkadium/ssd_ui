# ACCESSIBILITY NOTES

## Module
Portal

## Baseline

- GIGW/STQC/accessibility requirements must be considered from initial design.
- Accessibility evidence must be captured before release approval.

## Current Status

STARTED

## Notes

- Accessibility evidence is not fully captured yet.
- The first App Shell and Super Admin Dashboard start the baseline with semantic landmarks/headings, skip link support, visible focus styles, icon labels, keyboard-reachable buttons, and text summaries around charts.
- Profile, Preferences, Reminders, and Notifications pages use labelled headings, text status labels, table headers, and dialog semantics.
- Reminder/notification status is not color-only; badges include text labels.
- Manual keyboard, screen reader, contrast, and Hindi text expansion review remains pending before release approval.
- Modal focus trapping remains a future refinement once a standardized dialog primitive is approved/available.
